Connectgold Mobile Mediation SDK Integration Guide

Connectgold is a high-yield mobile ad mediation engine powered by Cloudflare Workers. Developers integrate Connectgold's SDK and adapters to connect 30 major mobile ad networks into a unified, AI-optimized mediation waterfall.

How Connectgold Mediation Works:
  • Dynamic Reflection Engine: The SDK uses Class.forName to dynamically discover and instantiate installed network adapters from a real-time Cloudflare Worker waterfall.
  • Zero Crashing (Graceful Failover): If an SDK dependency is omitted from an app build, the reflection engine catches ClassNotFoundException automatically and skips to the next available network in the waterfall.
  • 30 Integrated Networks: Enable high fill rates across global regions through built-in adapter classes.

1Gradle Setup (`app/build.gradle`)

Add the Connectgold Core SDK (which includes Connectgold's personal ad network adapter and all 30 built-in mediation adapter classes). Then, simply add the official Gradle dependencies for any of the 30 partner networks listed in Section 5: Connectgold Integrated Network Matrix that you want to enable in your mediation waterfall. Developers can use whichever SDK versions they prefer:

dependencies { // 1. Core Connectgold Mobile SDK (Includes Connectgold Core Adapter + 30 Integrated Network Adapters) implementation 'com.connectgold.sdk:connectgold-core:1.0.0' // 2. Add Partner Network SDKs from the 30-Network Matrix to enable them in your mediation waterfall // (Replace X.Y.Z with your preferred version for each SDK) implementation 'com.yandex.android:mobileads:X.Y.Z' // Yandex Ads implementation 'com.google.android.gms:play-services-ads:X.Y.Z' // Google AdMob implementation 'com.unity3d.ads:unity-ads:X.Y.Z' // Unity Ads implementation 'com.ironsource.sdk:mediationsdk:X.Y.Z' // ironSource implementation 'com.inmobi.monetization:inmobi-ads-kotlin:X.Y.Z' // InMobi implementation 'com.startapp:inapp-sdk:X.Y.Z' // Start.io // ... add any additional SDKs from the 30 Supported Network Matrix as needed }

2Android Manifest Configuration

Ensure standard network permissions are granted in AndroidManifest.xml to allow mediation ad networks to fetch ad creatives:

<manifest xmlns:android="http://schemas.android.com/apk/res/android"> <!-- Required Internet & Network State Permissions --> <uses-permission android:name="android.permission.INTERNET" /> <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" /> <application> <!-- CONNECTGOLD PUBLISHER ID --> <meta-data android:name="com.connectgold.sdk.PUBLISHER_ID" android:value="YOUR_PUBLISHER_ID" /> </application> </manifest>

3Format-Specific Java Implementation Guide

While the underlying waterfall engine is universal, how you request and handle ad objects in Java differs by ad format:

A. Banner Ad (Programmatic View Layout)

Banners return an android.view.View object that must be added programmatically to an existing layout container:

// 1. Programmatically create layout container in Java FrameLayout bannerContainer = new FrameLayout(this); setContentView(bannerContainer); String bannerUrl = "https://sweet-shape-f66a.trustconnect713.workers.dev/?platform=android&format=banner&pub_id=YOUR_PUB_ID"; ConnectgoldMediation.fetchAndLoad(this, bannerUrl, new AdLoadListener() { @Override public void onAdLoaded(String networkName, Object adView) { if (adView instanceof android.view.View) { bannerContainer.removeAllViews(); bannerContainer.addView((android.view.View) adView); } } @Override public void onAdFailedToLoad(String reason) { Log.e("Connectgold", "Banner load failed across waterfall: " + reason); } });

B. Fullscreen Interstitial Ad

Interstitials load in memory and are displayed over the current Activity using ConnectgoldMediation.showAd():

String interstitialUrl = "https://sweet-shape-f66a.trustconnect713.workers.dev/?platform=android&format=interstitial&pub_id=YOUR_PUB_ID"; ConnectgoldMediation.fetchAndLoad(this, interstitialUrl, new AdLoadListener() { @Override public void onAdLoaded(String networkName, Object adObject) { Log.d("Connectgold", "Interstitial ready from: " + networkName); // Show fullscreen overlay over current activity ConnectgoldMediation.showAd(MainActivity.this, adObject); } @Override public void onAdFailedToLoad(String reason) { Log.e("Connectgold", "Interstitial failed: " + reason); } });

C. Rewarded Video Ad (With User Reward Callback)

Rewarded ads trigger a reward callback listener when video playback completes:

String rewardedUrl = "https://sweet-shape-f66a.trustconnect713.workers.dev/?platform=android&format=rewarded&pub_id=YOUR_PUB_ID"; ConnectgoldMediation.fetchAndLoad(this, rewardedUrl, new AdLoadListener() { @Override public void onAdLoaded(String networkName, Object adObject) { // Pass a RewardListener to grant in-app coins/items upon video completion ConnectgoldMediation.showRewardedAd(MainActivity.this, adObject, new RewardListener() { @Override public void onUserRewarded(String rewardType, int amount) { Log.d("Connectgold", "Reward earned: " + amount + " " + rewardType); grantInAppCoins(amount); } }); } @Override public void onAdFailedToLoad(String reason) { Log.e("Connectgold", "Rewarded video failed: " + reason); } });

D. Native Ad (Custom View Card Binding)

Native ads return a structured object containing headline, body text, image, and CTA button:

String nativeUrl = "https://sweet-shape-f66a.trustconnect713.workers.dev/?platform=android&format=native&pub_id=YOUR_PUB_ID"; ConnectgoldMediation.fetchAndLoad(this, nativeUrl, new AdLoadListener() { @Override public void onAdLoaded(String networkName, Object nativeAdObj) { ConnectgoldNativeAd nativeAd = (ConnectgoldNativeAd) nativeAdObj; // Bind data programmatically into custom Java view TextView titleView = new TextView(MainActivity.this); titleView.setText(nativeAd.getTitle()); Button ctaButton = new Button(MainActivity.this); ctaButton.setText(nativeAd.getCtaText()); ctaButton.setOnClickListener(v -> nativeAd.performClick()); LinearLayout nativeLayout = new LinearLayout(MainActivity.this); nativeLayout.setOrientation(LinearLayout.VERTICAL); nativeLayout.addView(titleView); nativeLayout.addView(ctaButton); setContentView(nativeLayout); } @Override public void onAdFailedToLoad(String reason) { Log.e("Connectgold", "Native ad failed: " + reason); } });

E. VAST Video Stream XML

For custom video players (ExoPlayer / MediaPlayer), pass format=vast to retrieve raw VAST XML streams:

String vastUrl = "https://sweet-shape-f66a.trustconnect713.workers.dev/?platform=android&format=vast&pub_id=YOUR_PUB_ID"; ConnectgoldMediation.fetchVastStream(this, vastUrl, new VastStreamListener() { @Override public void onVastReady(String vastXmlUrl) { // Pass VAST URL to ExoPlayer MediaItem exoPlayer.setMediaItem(MediaItem.fromUri(vastXmlUrl)); exoPlayer.prepare(); } @Override public void onVastFailed(String error) { Log.e("Connectgold", "VAST stream failed: " + error); } });

4Writing Custom Mediation Adapters

Developers can extend Connectgold mediation by creating custom adapter classes implementing the ConnectgoldAdapter interface:

Step A: Define the Adapter Interface

package com.connectgold.sdk.adapters; import android.content.Context; import org.json.JSONObject; public interface ConnectgoldAdapter { interface Listener { void onSuccess(String networkName, Object adView); void onFailure(String errorReason); } void loadAd(Context context, JSONObject params, Listener listener); }

Step B: Sample Custom Adapter Implementation

package com.yourcompany.app.adapters; import android.content.Context; import org.json.JSONObject; import com.connectgold.sdk.adapters.ConnectgoldAdapter; public class CustomAdNetworkAdapter implements ConnectgoldAdapter { @Override public void loadAd(Context context, JSONObject params, Listener listener) { String placementId = params.optString("placement_id"); String appId = params.optString("app_id"); ThirdPartySDK.requestAd(context, appId, placementId, new ThirdPartySDK.Callback() { @Override public void onAdReady(Object view) { listener.onSuccess("CustomAdNetwork", view); } @Override public void onError(String error) { listener.onFailure(error); } }); } }

7GDPR, EEA & Regional Privacy Consent

Pass regional user consent status in your worker request parameters:

  • &consent=1 : User granted consent for personalized ads.
  • &consent=0 : User denied consent (Serves non-personalized ads).
